# UI Guidelines

KUTT debe sentirse premium, sobrio y rapido. La interfaz sirve a la operacion diaria, no al lucimiento visual.

## Principios

- Mobile first.
- Interacciones simples.
- Navegacion inferior para acciones frecuentes.
- Estetica premium sobria.
- Futuro sistema visual KUTT Liquid UI.
- Transparencias solo si mejoran jerarquia y lectura.

## Mobile first

La mayoria de acciones deben funcionar comodo desde un telefono:

- Cobrar.
- Crear cita.
- Registrar ingreso o egreso.
- Cerrar dia.
- Buscar cliente.

Si una interaccion es lenta en movil, no esta lista.

## Navegacion

La navegacion principal debe priorizar:

- Hoy.
- Agenda.
- Caja.
- Clientes.
- Mas / Ajustes.

La navegacion por swipe puede evaluarse mas adelante si mejora velocidad sin esconder acciones importantes.

## KUTT Liquid UI

KUTT Liquid UI puede inspirarse en glassmorphism e interfaces tipo iOS, pero debe respetar:

- Contraste suficiente.
- Texto legible.
- Jerarquia clara.
- Estados tactiles evidentes.
- Buen rendimiento en movil.

Las transparencias, blur y capas liquidas son recursos de jerarquia, no decoracion gratuita.
